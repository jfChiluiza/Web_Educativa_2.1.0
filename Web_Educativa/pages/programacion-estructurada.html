¿Qué es la Programación Estructurada?

La programación estructurada es un paradigma de programación que busca mejorar la claridad, calidad y tiempo de desarrollo utilizando únicamente tres estructuras básicas:

Secuencia

Ejecución de acciones una después de otra

Selección

Ejecución de una u otra acción según una condición (if-else)

Iteración

Repetición de una acción mientras se cumpla una condición (bucles)

Ejemplo Práctico
// Secuencia
nombre = input("Tu nombre")
edad = input("Tu edad")

// Selección
if edad >= 18:
    print("Eres mayor de edad")
else:
    print("Eres menor de edad")

// Iteración
for i in range(3):
    print("¡Hola " + nombre + "!")
Beneficios clave:
  • Código más fácil de leer y mantener
  • Reducción de la complejidad del programa
  • Mayor facilidad para detectar y corregir errores
  • Base fundamental para aprender otros paradigmas

¿Qué es una Variable?

Una variable es un espacio en la memoria del computador que almacena datos que pueden cambiar durante la ejecución del programa. Podemos pensarla como una "caja" con un nombre, donde guardamos información.

Características de las variables:
  • Tienen un nombre único (identificador)
  • Almacenan un valor específico
  • Su contenido puede cambiar durante la ejecución
  • Tienen un tipo de dato asociado
Tipos de Variables Básicos
Números
# Enteros (int)
edad = 25
cantidad = -10

# Decimales (float)
precio = 19.99
temperatura = -3.5
Texto (String)
# Cadenas de texto
nombre = "Ana"
mensaje = 'Hola Mundo'
direccion = "Calle Principal #123"
Booleanos
# Valores verdadero/falso
es_estudiante = True
tiene_permiso = False
Listas
# Colecciones de datos
notas = [85, 92, 78, 95]
nombres = ["Juan", "Ana", "Carlos"]
Ejemplos Prácticos
Calculadora de Promedio
# Declaración de variables
nota1 = 85
nota2 = 92
nota3 = 78

# Cálculo del promedio
promedio = (nota1 + nota2 + nota3) / 3

# Mostrar resultado
print(f"El promedio es: {promedio}")
Información Personal
# Datos personales
nombre = "María García"
edad = 28
altura = 1.65
es_estudiante = True

# Mostrar información
print(f"Nombre: {nombre}")
print(f"Edad: {edad} años")
print(f"Altura: {altura} metros")
print(f"¿Es estudiante? {es_estudiante}")
Tips para nombrar variables:
  • Usa nombres descriptivos (ej: edad_usuario es mejor que e)
  • Comienza con letra minúscula
  • No uses espacios (usa guion bajo: nombre_completo)
  • Evita caracteres especiales y palabras reservadas

¿Qué son los Tipos de Datos?

Los tipos de datos definen el tipo de valor que puede almacenar una variable y las operaciones que se pueden realizar con ella. Cada tipo de dato ocupa un espacio diferente en memoria y tiene sus propias características.

Tipos de Datos Fundamentales
Tipos Numéricos
  • int (enteros): -3, 0, 42
  • float (decimales): 3.14, -0.001
  • double (decimales precisos): 3.141592653589793
edad = 25          # int
peso = 68.5       # float
pi = 3.14159265  # double
Tipos de Texto
  • char (carácter): 'A', '1', '$'
  • string (cadena): "Hola Mundo"
letra = 'A'           # char
nombre = "Juan"      # string
texto = '''Múltiples
líneas'''           # string
Tipos Lógicos
  • boolean: True/False
es_mayor = True
tiene_permiso = False
resultado = 5 > 3  # True
Tipos Compuestos
  • list: [1, 2, 3]
  • tuple: (1, 2, 3)
  • dict: {"clave": "valor"}
lista = [1, 2, 3]
tupla = (1, "dos", True)
diccionario = {"nombre": "Ana"}
Conversión entre Tipos de Datos
Ejemplos de Conversión
# Conversión de string a int
edad_texto = "25"
edad_numero = int(edad_texto)    # 25

# Conversión de número a string
precio = 19.99
precio_texto = str(precio)       # "19.99"

# Conversión a boolean
numero = 1
booleano = bool(numero)         # True

# Conversión entre tipos numéricos
decimal = float(5)              # 5.0
entero = int(5.7)              # 5
Consideraciones Importantes:
  • Cada tipo de dato tiene sus propias operaciones permitidas
  • La conversión entre tipos puede causar pérdida de datos
  • No todos los tipos son compatibles para conversión directa
  • Es importante validar antes de realizar conversiones

Flujo de control en programación
¿Qué son las Estructuras de Control?

Las estructuras de control son herramientas que nos permiten controlar el flujo de ejecución de un programa, permitiéndonos tomar decisiones y repetir acciones según ciertas condiciones.

Estructuras Condicionales
If-Else
edad = 18

if edad >= 18:
    print("Eres mayor de edad")
elif edad >= 13:
    print("Eres adolescente")
else:
    print("Eres menor de edad")
Switch/Match (Python 3.10+)
dia = "lunes"

match dia:
    case "lunes":
        print("Inicio de semana")
    case "viernes":
        print("¡Fin de semana!")
    case _:
        print("Día entre semana")
Operador Ternario
# Forma tradicional
resultado = "Aprobado" if nota >= 60 else "Reprobado"
Estructuras de Repetición
Bucle For
# Iteración sobre rango
for i in range(5):
    print(i)  # Imprime 0,1,2,3,4

# Iteración sobre lista
frutas = ["manzana", "pera", "uva"]
for fruta in frutas:
    print(fruta)
Bucle While
contador = 5
while contador > 0:
    print(contador)
    contador -= 1
Ejemplos Prácticos
Calculadora Simple
operacion = input("Operación (+,-,*,/): ")
num1 = float(input("Primer número: "))
num2 = float(input("Segundo número: "))

match operacion:
    case "+":
        resultado = num1 + num2
    case "-":
        resultado = num1 - num2
    case "*":
        resultado = num1 * num2
    case "/":
        if num2 != 0:
            resultado = num1 / num2
        else:
            resultado = "Error: División por cero"
    case _:
        resultado = "Operación no válida"

print(f"Resultado: {resultado}")
Juego de Adivinanza
numero_secreto = 42
intentos = 0

while True:
    intento = int(input("Adivina el número (1-100): "))
    intentos += 1
    
    if intento == numero_secreto:
        print(f"¡Correcto! Lo lograste en {intentos} intentos")
        break
    elif intento < numero_secreto:
        print("El número es mayor")
    else:
        print("El número es menor")
Buenas Prácticas:
  • Usa nombres descriptivos para las variables de control
  • Evita anidar demasiados niveles de condicionales
  • Incluye siempre una condición de salida en bucles while
  • Considera usar 'break' y 'continue' cuando sea necesario
  • Mantén el código dentro de los bloques simple y legible

Programación Orientada a Objetos: Clases y Objetos

Clases y Objetos: Bloques fundamentales de la POO
¿Qué son las Clases y Objetos?
Clase

Una clase es una plantilla o molde que define las propiedades (atributos) y comportamientos (métodos) que tendrán los objetos de ese tipo. Es como un plano para crear objetos.

Objeto

Un objeto es una instancia de una clase. Es una entidad concreta creada a partir de la plantilla (clase) que tiene sus propios valores para los atributos definidos.

Ejemplos Prácticos
Ejemplo: Clase Coche
# Definición de la clase Coche
class Coche:
    # Constructor: se ejecuta al crear un nuevo objeto Coche
    # Inicializa los atributos básicos del coche
    def __init__(self, marca, modelo, año):
        # Atributos del coche
        self.marca = marca      # Marca del coche (ej: Toyota)
        self.modelo = modelo    # Modelo del coche (ej: Corolla)
        self.año = año         # Año de fabricación
        self.encendido = False # Estado del motor (apagado por defecto)
    
    # Método para encender el coche
    # Cambia el estado del coche a encendido y retorna un mensaje
    def encender(self):
        self.encendido = True
        return f"{self.marca} {self.modelo} encendido"
    
    # Método para obtener la información del coche
    # Retorna un string con los detalles principales
    def info(self):
        return f"Coche {self.marca} {self.modelo} ({self.año})"

# Creación de objetos (instancias) de la clase Coche
mi_coche = Coche("Toyota", "Corolla", 2022)    # Primer coche
otro_coche = Coche("Honda", "Civic", 2023)     # Segundo coche

# Uso de los métodos del objeto
print(mi_coche.info())      # Muestra: Coche Toyota Corolla (2022)
print(mi_coche.encender())  # Muestra: Toyota Corolla encendido
Ejemplo: Cuenta Bancaria
# Definición de la clase CuentaBancaria
class CuentaBancaria:
    # Constructor: inicializa una nueva cuenta bancaria
    # Parámetros:
    # - titular: nombre del dueño de la cuenta
    # - saldo: cantidad inicial (opcional, por defecto 0)
    def __init__(self, titular, saldo=0):
        self.titular = titular  # Nombre del titular de la cuenta
        self.saldo = saldo     # Saldo actual de la cuenta
    
    # Método para depositar dinero en la cuenta
    # Parámetros:
    # - cantidad: monto a depositar
    # Retorna: mensaje con el nuevo saldo
    def depositar(self, cantidad):
        self.saldo += cantidad  # Aumenta el saldo
        return f"Depósito de ${cantidad}. Nuevo saldo: ${self.saldo}"
    
    # Método para retirar dinero de la cuenta
    # Parámetros:
    # - cantidad: monto a retirar
    # Retorna: mensaje de éxito o error si no hay suficiente saldo
    def retirar(self, cantidad):
        # Verifica si hay suficiente saldo
        if cantidad <= self.saldo:
            self.saldo -= cantidad  # Reduce el saldo
            return f"Retiro de ${cantidad}. Nuevo saldo: ${self.saldo}"
        return "Saldo insuficiente"

# Ejemplo de uso de la clase
# Crear una nueva cuenta con saldo inicial de $1000
cuenta = CuentaBancaria("Juan Pérez", 1000)

# Realizar operaciones en la cuenta
print(cuenta.depositar(500))  # Depositar $500
print(cuenta.retirar(200))    # Retirar $200
Buenas Prácticas:
  • Los nombres de las clases deben comenzar con mayúscula
  • Cada clase debe tener una única responsabilidad
  • Utiliza métodos para modificar los atributos (encapsulamiento)
  • Documenta tus clases y métodos
  • Mantén las clases cohesivas y con bajo acoplamiento

¿Qué es la Herencia?

La herencia es un mecanismo que permite crear nuevas clases basadas en clases existentes, heredando sus atributos y métodos. Esto permite:

  • Reutilización de código
  • Establecer jerarquías de clases
  • Extender funcionalidades existentes
  • Crear código más mantenible
Ejemplo: Jerarquía de Vehículos
# Clase base (padre)
class Vehiculo:
    def __init__(self, marca, modelo, año):
        self.marca = marca
        self.modelo = modelo
        self.año = año
        self.encendido = False
    
    def encender(self):
        self.encendido = True
        return f"{self.marca} {self.modelo} encendido"
    
    def info(self):
        return f"Vehículo {self.marca} {self.modelo} ({self.año})"

# Clase derivada (hijo)
class Coche(Vehiculo):
    def __init__(self, marca, modelo, año, num_puertas):
        # Llamada al constructor de la clase padre
        super().__init__(marca, modelo, año)
        self.num_puertas = num_puertas
        self.velocidad = 0
    
    def acelerar(self, kmh):
        if self.encendido:
            self.velocidad += kmh
            return f"Acelerando a {self.velocidad} km/h"
        return "Primero debes encender el coche"

# Otra clase derivada
class Motocicleta(Vehiculo):
    def __init__(self, marca, modelo, año, tipo):
        super().__init__(marca, modelo, año)
        self.tipo = tipo
        self.caballete_puesto = False
    
    def poner_caballete(self):
        self.caballete_puesto = True
        return "Caballete puesto"

# Uso de las clases
mi_coche = Coche("Toyota", "Corolla", 2022, 4)
mi_moto = Motocicleta("Honda", "CBR", 2023, "Deportiva")

# Usando métodos heredados
print(mi_coche.encender())  # Método heredado
print(mi_coche.acelerar(50))  # Método propio

print(mi_moto.info())  # Método heredado
print(mi_moto.poner_caballete())  # Método propio

Polimorfismo: Diferentes formas de un mismo comportamiento
Interfaces comunes, implementaciones diferentes
Múltiples comportamientos, misma interfaz
¿Qué es el Polimorfismo?

El polimorfismo es la capacidad de un objeto para tomar diferentes formas. Permite que:

  • Diferentes objetos respondan al mismo método de diferentes maneras
  • Se puedan tratar objetos de diferentes clases de manera uniforme
  • El código sea más flexible y reutilizable
  • Se implementen interfaces comunes con comportamientos específicos
Ejemplo Práctico: Sistema de Empleados
# Clase base para todos los empleados
class Empleado:
    def __init__(self, nombre, id):
        self.nombre = nombre
        self.id = id
    
    def calcular_salario(self):
        # Método que será sobrescrito por las clases hijas
        pass
    
    def obtener_rol(self):
        # Método que será sobrescrito por las clases hijas
        pass
    
    def info(self):
        return f"Empleado: {self.nombre} (ID: {self.id})"

# Clase para empleados con salario fijo
class EmpleadoFijo(Empleado):
    def __init__(self, nombre, id, salario_mensual):
        super().__init__(nombre, id)
        self.salario_mensual = salario_mensual
    
    def calcular_salario(self):
        return self.salario_mensual
    
    def obtener_rol(self):
        return "Empleado Fijo"

# Clase para empleados por horas
class EmpleadoPorHoras(Empleado):
    def __init__(self, nombre, id, horas_trabajadas, tarifa_por_hora):
        super().__init__(nombre, id)
        self.horas_trabajadas = horas_trabajadas
        self.tarifa_por_hora = tarifa_por_hora
    
    def calcular_salario(self):
        return self.horas_trabajadas * self.tarifa_por_hora
    
    def obtener_rol(self):
        return "Empleado Por Horas"

# Clase para vendedores con comisión
class Vendedor(Empleado):
    def __init__(self, nombre, id, ventas_totales, porcentaje_comision):
        super().__init__(nombre, id)
        self.ventas_totales = ventas_totales
        self.porcentaje_comision = porcentaje_comision
    
    def calcular_salario(self):
        return self.ventas_totales * (self.porcentaje_comision / 100)
    
    def obtener_rol(self):
        return "Vendedor"

# Función que demuestra el polimorfismo
def mostrar_detalles_empleado(empleado):
    print(f"\nDetalles de {empleado.obtener_rol()}:")
    print(empleado.info())
    print(f"Salario: ${empleado.calcular_salario():.2f}")

# Crear diferentes tipos de empleados
empleados = [
    EmpleadoFijo("Juan Pérez", "E001", 3000),
    EmpleadoPorHoras("Ana García", "E002", 120, 15),
    Vendedor("Carlos López", "E003", 50000, 10)
]

# Procesar todos los empleados de manera uniforme
for empleado in empleados:
    mostrar_detalles_empleado(empleado)

# Calcular nómina total
nomina_total = sum(emp.calcular_salario() for emp in empleados)
print(f"\nNómina total: ${nomina_total:.2f}")

# Ejemplo de salida:
"""

Salario: $3000.00

Empleado: Ana García (ID: E002)
Salario: $1800.00

Detalles de Vendedor:

Explicación del Polimorfismo en el Ejemplo
  • Clase Base

    La clase Empleado define la interfaz común con métodos que todas las clases hijas deben implementar.

  • Clases Derivadas

    Cada tipo de empleado implementa los métodos de manera diferente según sus necesidades específicas.

  • Polimorfismo en Acción

    La función mostrar_detalles_empleado() trabaja con cualquier tipo de empleado sin necesidad de saber su clase específica.

  • Tratamiento Uniforme

    Podemos procesar una lista de diferentes tipos de empleados de manera uniforme, demostrando la flexibilidad del polimorfismo.
